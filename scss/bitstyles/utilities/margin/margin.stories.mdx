# Margin

Apply margins of a specified size and direction to an element.

With default configuration, the margins are available in sizes `0`, `s7`, `s6`, `s5`, `s4`, `s3`, `s2`, `s1`, `m`, `l1`, `l2`, `l3`, `l4`, `l5`, `l6`, `l7`, `auto`, with negative-margin variations of each (except `auto`). They are provided at the `s` and `m` breakpoints. See [customization](#customization) below for details on how to change that.

## Customization

Available margin-sizes can be configured by overriding the `$sizes` Sass map. Provide the name to be used for the class as the key, and the value for the property as the value. If you want to keep the existing values, youâ€™ll need to copy the default configuration as you cannot merge or update the existing map.

The `positive` and `negative` keys are required for each group you want to define (e.g. if you have no negative margins, that key is not required).

```scss
@use '~bitstyles/scss/bitstyles/utilities/margin' with (
  $sizes: (
    'positive': (
      '0': 0,
      's6': var(design-token.get('size', 's6')),
      's4': var(design-token.get('size', 's4')),
      's': var(design-token.get('size', 's2')),
      'm': var(design-token.get('size', 'm')),
      'l': var(design-token.get('size', 'l2')),
      'l3': var(design-token.get('size', 'l3')),
      'auto': auto,
    ),
    'negative': (
      's6': calc(var(design-token.get('size', 's6')) * -1),
      's4': calc(var(design-token.get('size', 's4')) * -1),
      's': calc(var(design-token.get('size', 's2')) * -1),
      'm': calc(var(design-token.get('size', 'm')) * -1),
      'l': calc(var(design-token.get('size', 'l2')) * -1),
      'l3': calc(var(design-token.get('size', 'l3')) * -1),
    ),
  )
);
```

To customize the breakpoints these classes are available at, override `$breakpoints` e.g.

```scss
@use '~bitstyles/scss/bitstyles/utilities/margin' with (
  $breakpoints: (
    's',
    'm',
  )
);
```

Directions can be customized by overriding the `$directions` Sass map. Provide the name for the direction as the key, and the list of directions. Note that if you want to keep the existing directions, youâ€™ll need to copy the existing configurations â€” the map cannot be updated, only overwritten.

```scss
@use '~bitstyles/scss/bitstyles/utilities/margin' with (
  $directions: (
    '': null,
    'top-and-right': (
      'top',
      'right',
    ),
  )
);
```

The `'': null` entry is needed to render the margin classes with no directional modifiers e.g. `u-margin-m`, `u-margin-l`.
