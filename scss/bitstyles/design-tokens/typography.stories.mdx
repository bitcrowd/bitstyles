# Typography

The typographic scale is independent of the HTML element you may use for the content in question — `u-text-xs` can be applied to a `<span>` just as well as a `<h1>`, for example. This allows you freedom to use the headings in correct semantic order when the visual design you’re building doesn’t match the logically-dictated order of your heading structure.

## Customization

The typographic scale can be customized by overriding the base font-size, and the ratio used to calculate each successive font-size:

```scss
$bitstyles-font-size-base-small: 16px;
$bitstyles-font-size-base: 18px;
$bitstyles-font-ratio: 1.2;
```

Note that while these are specified in `px` units, the output CSS will use the equivalent `rem`, for accessibility reasons.

You can also customize the names of the font-sizes (e.g. `xl`, `large`, `5`), and their point on the typographic scale. With this you can skip points of the scale, or place a point at a non-integer position, allowing you to distort the linear ratio if needed.

If the designs you’re implementing follow a different typographic scale — one not linearly-computable with a ratio/multiplier — you can instead manually specify the font-sizes at every point, and for each breakpoint. By default, the scale is altered to better-fit screensizes at the `m` breakpoint, but you can use any of the defined breakpoints:

```scss
$bitstyles-font-sizes: (
  'base': (
    'xxxl': 32px,
    'xxl': 24px,
    'xl': 20px,
    'l': 18px,
    'm': $bitstyles-font-size-base-small,
    's': $bitstyles-font-size-base-small,
    'xs': 16px
  ),
  'm': (
    'xxxl': 86px,
    'xxl': 40px,
    'xl': 28px,
    'l': 20px,
    'm': $bitstyles-font-size-base,
    's': $bitstyles-font-size-base,
    'xs': 16px
  )
);
```
