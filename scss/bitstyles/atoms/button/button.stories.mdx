## Buttons

Bitstyles provides a set of button variants divided around two aspects: the colors used, and the size and shape of the button. These can be mixed to produce a range of buttons that conform to standard colors on hover, active, and pressed or selected states, and whose sizes will fit when placed in a row or column next to other variants.

The following example use the default configuration, see [customization](#customization) for information on how to change these.

Available color variants:

|Name|Classname|Usage|
|Default/Primary|`a-button`|A highly-visible button. Use it for the primary action on a page|
|Secondary|`.a-button--secondary`|General-purpose button. Use it for common actions|
|Reversed|`.a-button--reversed`|General-purpose button, for use on dark backgrounds|
|Danger|`.a-button--danger`|A modifier color variant that has no default colors of its own, only hover/focus colors. Apply this to any other color variant and it’ll turn red on hover, indicating to the user that the action this button performs is destructive, not undoable, or somehow dangerous|
|Transparent|`.a-button--transparent`|A button that has no appearance until hovered|
|Tab|`.a-button--tab`||

All the buttons produce a larger-than-text hit target, and by default react to `:hover`, `:focus`, `:active`, and `:disabled`. Visited states are not shown even if the button is an anchor, as “visited” doesn’t make sense for an action.

## Buttons with icons and text

**Note**: if you want a button with _only_ an icon, go look at [`Icon buttons`](/docs/atoms-button-icon--button-element)

These are buttons containing a visible text label next to an icon. The order of text/icon is dictated by the order of the elements, you do not need to add a class to change it.

  <Story name="Button with Icon">
