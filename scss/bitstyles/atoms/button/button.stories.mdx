# Buttons

The button classes are used to make an interactive element look like a “button” to the user, and should ideally perform an action rather a navigation (a call-to-action button is a common exception to this though). For historical and technical reasons, things which we consider “actions” are sometimes implemented by navigating to a page (e.g. in a CRUD system) so this class can be applied to `<a>` elements. Ideally though (and definitely if you’re working with JS functionality) use a real `<button>` element.

All the buttons produce a larger-than-text hit target, and by default react to `:hover`, `:focus`, `:active`, `:disabled`, and `aria-pressed`, `aria-selected`, and `aria-current` states. Visited states are not shown even if the “button” is actually an anchor element (this is because buttons indicate actions, and “visited” doesn’t make sense for an action).

Bitstyles provides buttons with two axis of variance — shape variants, and color variants. Each of the size variants can be combined with each of the color variants to produce a wide range of button appearances.

## Shape variants

Buttons are available in a variety of shapes:

- **small**, **default**, **large** Sizes which can be combined with the other shape and color variants.
- **square** A square button, with padding and border-radius that matches the other buttons at a given size. Useful for icon buttons.
- **round** A button with rounded corners, which matches the size of other buttons at a given size. Combine with **square** to make a circular button (useful for circular icon buttons, if that’s your design style).
- **tab** For use in a tabbed interface, these are a combination of shape & color variant, to create an appearance unlike other buttons.

See [Button shapes](/docs/atoms-button-shapes--page) for details on customizing these.
See [Tab buttons](/docs/atoms-button-tabs--page) for details on the Tab buttons.

## Color variants

- **primary** The default button style is `primary`, and in most cases should be used only once per screen to indicate the primary action for the screen.
- **secondary** The most-commonly used, `secondary` buttons indicate all the non-primary actions a user can enact on a screen.
- **transparent** Use `transparent` buttons to de-emphasize an action (see the “cancel” buttons in the examples up top). Use also for nav menus, sidebars, and dropdown menus.
- **outline** Use `outline` buttons to de-emphasize an action (see the “cancel” buttons in the examples up top). Use also for nav menus, sidebars, and dropdown menus.
- **danger** A special case that only applies its appearance on hover and active. Use these to indicate an action the user should be wary of, most often something destructive to data. This variant can be combined with all other button color variants.
- **tab** For use in a tabbed interface, these are a combination of shape & color variant, to create an appearance quite different to the other buttons.

See [Button colors](/docs/atoms-button-colors--page) for details on customizing these.
See [Tab buttons](/docs/atoms-button-tabs--page) for details on the Tab buttons.

### Disabled buttons

Use the `aria-disabled="true"` attribute on `anchor` elements, or the `disabled` attribute on `button` elements.

## Pressed, selected, current

Buttons can be given aria attributes to indicate to the user that they have a another state. This normally gives the user a little more information about how that button is intended to be used, or the effects it will have when clicked.

Apply `aria-pressed="true"` to your button to indicate that it’s a toggle button in a pressed state.

Apply `aria-selected="true` to the currently-selected tab button in a series of tab buttons.

Apply `aria-current="page"` to the navigation button in your main navigation, when the user is currently on that page.
