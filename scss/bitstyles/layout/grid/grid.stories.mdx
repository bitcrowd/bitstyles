# Grid Systems

A responsive html grid system using inline-block.

Note that for technical reasons, **whitespace is important**. If necessary, use html comments between each `.l-grid__item` to remove any whitespace (see the markup example below), otherwise the browser will display a small gap between grid items.

This is an HTML grid system (not a CSS Grid system), so some wrapping elements are needed to create rows and columns (see the markup in the examples below). Each of these grid items needs the class `.l-grid__item` and almost always a width class (if you do not specify a width, the grid item will default to 100% width). With current defaults, the widths provided include a 12-column system e.g. 1-of-12, 2-of-12, 3-of-12 … 11-of-12, 12-of-12.

Grids can be nested.

### Dependencies

- `layout/width`

## Grid system — base

In the base grid system, columns are of proportional widths (a percentage of the full width) but not responsive — that percentage doesn’t change when the viewport does.

This means they cannot change from being 3-column on larger screens, to one  stacked column on small screens (if you can, try resizing your browser window to see that the number of columns in the example below doesn’t change). See the Responsive Grid below for information on how to make your grid responsive.

## Grid system - responsive

The grid system can be made responsive, allowing you to specify different widths for an element, based on the viewport size.

To do this, append the name of a breakpoint to the width class to have it activate only at that breakpoint e.g. `.l-width--12-of-12.l-width--6-of-12@m` creates an element that’s fullwidth for all viewports, then becomes half width for `m`-sized viewports and larger. This is commonly used to create a layout of 100%-width grid items stacked vertically on smaller viewports, changing to e.g. a three-column layout on larger viewport sizes.

Multiple width/breakpoint combinations can be used to define a grid item’s width at several breakpoints (for all the breakpoints defined in the `$bitstyles-widths-breakpoints` Sass list).
